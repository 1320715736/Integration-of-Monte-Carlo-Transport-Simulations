# 文献工具流总结

文献来源：`Integration of Monte Carlo Transport Simulations into a TCAD Workflow for Electron Detector Developments`

这份文档按论文原始方法，把从 Monte Carlo 到 Sentaurus TCAD 的工具流完整拆开，方便后续在本项目里对应实现 Python 部分。

## 1. 目标

论文的目标不是直接在 TCAD 里做粒子输运，而是把 Monte Carlo 软件给出的电子空间分布，转换成 Sentaurus 可以读取的 `OpticalGeneration` 激励文件，再交给 `Sdevice` 做器件电学仿真。

核心思想是两步法：

1. 先把离散粒子位置累积到一个均匀网格的“中间图像”里。
2. 再把这个中间图像插值到 Sentaurus 的非均匀网格节点上。

## 2. 总体工具流

### Step 1：建立器件结构与网格

- 工具：`Sprocess + SDE`
- 当前项目实际流程：先在 `Sprocess` 中完成工艺与结构构建，再转换到 `SDE`，随后在 `SDE` 中完成 2D 网格与用户字段处理
- 输入：工艺流程、PiN 结构、掺杂分布、器件尺寸
- 处理：
  - 在 `Sprocess` 中生成器件结构与掺杂结果
  - 将 `Sprocess` 结果转换到 `SDE`
  - 在 `SDE` 中保持 2D 结构并生成 Sentaurus 网格
  - 在需要导入外部生成率的半导体区域加入 `PMIUserField0`、`PMIUserField1`
- 输出：带用户字段的 2D Sentaurus 网格数据

论文里这样做的目的，是让后续脚本能够获得网格坐标信息，并为后面写入 `OpticalGeneration` 做准备。

对当前项目来说，这一步应理解为：

- 结构主模型来源于 `Sprocess`
- `SDE` 的作用主要是承接结构、生成 2D 网格，并加入后续 Python/TDX 需要读取的辅助字段
- 按当前项目现有代码，`SDE` 阶段重点是生成 `PMIUserField0/1` 占位字段；`OpticalGeneration` 是在后续 Python 重写 `.dat` 时新增的

对当前项目现有代码，`PMIUserField0/1` 的使用原则是：

- 不是只放在某一条线或某一个局部窗口
- 而是作为覆盖目标半导体区域的占位字段
- 当前实现里，它们可以是覆盖整个硅区的零值字段
- 后续 Python 不依赖 `PMIUserField0/1` 的数值本身取坐标，而是直接从 `.grd` 读取顶点坐标
- 一般应覆盖整个 PiN 的硅区，至少覆盖电子束可能沉积并且后续需要参与电学求解的区域
- 不应放在金属、电极、气体、氧化层、钝化层等不需要写入光学生成率的区域

### Step 2：导出可读网格文件

- 工具：`TDX`
- 输入：Sentaurus 二进制网格文件
- 处理：
  - 把原始二进制网格转换成外部脚本可读的 `DF-ISE` 文本格式
- 输出：可供 Python 脚本读取的 DF-ISE 文件

这一步是 Python 和 Sentaurus 数据对接的入口。

补充说明：

- 对当前项目这条 Python 实现路线，`DF-ISE` 文本输入是需要的，因为后续脚本直接读取 `.grd/.dat`
- 如果改成完全沿用原论文附录里的 `TDX Tcl API` 直接在 `.tdr` 上操作，则不一定必须先导出 `DF-ISE` 文本

### Step 3：运行 Monte Carlo 粒子输运

- 工具：`Casino`
- 可替代工具：`Geant4`
- 输入：
  - 入射电子能量
  - 入射电子数
  - 材料参数
  - 几何结构
- 处理：
  - 模拟初级和次级电子在材料中的输运与沉积
- 输出：
  - 粒子位置数据文件
  - 每个电子级联的空间分布

论文中主要使用 `Casino`，但明确指出该流程同样可以扩展到 `Geant4`。

### Step 4：读取 Monte Carlo 输出并构建中间图像

- 工具：`Python`
- 输入：Monte Carlo 粒子位置文件
- 处理：
  - 定义一个均匀分辨率的规则网格，也就是 `intermediate picture`
  - 这个网格在横向和纵向上都使用固定步长
  - 网格横向尺寸要大于器件尺寸
  - 网格深度要大于电子最深穿透位置
  - 对每一个粒子位置，找到它在中间图像中的网格单元
  - 对对应网格单元做计数累积
  - 最后乘上 `optical generation factor`
- 输出：
  - 一个规则均匀网格上的粒子分布数组
  - 单位转换后的光学生成率分布

论文指出：

- 中间图像使用均匀网格是为了降低计算量
- 如果直接对每个 Sentaurus 网格点扫描所有粒子，耗时会很高
- 网格步长需要在精度和效率之间折中
- 文中认为 `200 nm` 是 2D 情况下一个较合适的折中值

### Step 5：读取 Sentaurus 网格并完成逐顶点插值

- 工具：`Python + scipy`
- 核心函数：`scipy.interpolate.interpn`
- 输入：DF-ISE 网格文件
- 处理：
  - 读取所有区域名
  - 识别需要写入光学生成的位置
  - 在当前项目实现中，直接从 `.grd` 读取顶点坐标
  - 建立 Sentaurus 网格点坐标列表
  - 将 Step 4 的中间图像通过 `interpn` 插值到 Sentaurus 每一个相关网格点
- 输出：
  - Sentaurus 网格节点坐标
  - 与每个网格点一一对应的 `OpticalGeneration` 数值
  - 区域名、轴编号、数据点编号等映射关系

按当前项目现有代码，这一步对应输出目录 `step5_output`。

### Step 6：将结果写入新的 `.dat`

- 工具：`Python`
- 输入：
  - 原始 DF-ISE `.dat`
  - Step 5 得到的逐顶点 `OpticalGeneration` 数组
- 处理：
  - 在保持原始 `.dat` 不变的前提下，新建一个输出 `.dat`
  - 在新 `.dat` 的 header 中加入 `OpticalGeneration`
  - 将 Step 5 的插值结果按顶点顺序写入 `Dataset ("OpticalGeneration")`
- 输出：
  - 带 `OpticalGeneration` 的新 `.dat` 文件

按当前项目现有代码，这一步对应输出目录 `step6_output`。

### Step 7：重新打包为 Sentaurus 原生格式

- 工具：`TDX`
- 输入：原始 `.grd` 与新生成的 `.dat`
- 处理：
  - 将文本数据重新转换回 Sentaurus 原生二进制格式
- 输出：
  - `.tdr` 光学生成激励文件

到这里，Monte Carlo 输出已经被“伪装”为 Sentaurus 可直接读取的外部光激励源。

### Step 8：在 Sdevice 中进行器件仿真

- 工具：`Sentaurus Device (Sdevice)`
- 输入：
  - 器件结构
  - 偏置条件
  - 时间积分参数
  - 上一步生成的 `OpticalGeneration` `.tdr`
- 处理：
  - 求解泊松方程和载流子连续性方程
  - 计算电子空穴对生成后的器件响应
- 输出：
  - 电荷积分结果
  - 点扩散函数 `PSF`
  - 调制传递函数 `MTF`
  - 其他器件性能指标

## 3. 为什么论文采用“两步法”

论文明确说明不直接“对每个 Sentaurus 节点扫描所有粒子”，原因是太慢。

采用两步法后的计算逻辑是：

1. 先把所有粒子只扫描一次，累积到中间图像。
2. 再从中间图像读取插值结果，而不是反复扫描原始粒子点。

这样做的好处：

- 大幅降低计算量
- 更适合集成到 Sentaurus Workbench
- 对 2D 问题速度很快

论文给出的量级是：

- 从 `10^6` 个电子位置构建 2D 中间图像约需 `1 min`
- 再生成 optical generation 约需 `40 s`
- 当时器件结构约 `97000` 个点

## 4. 论文里对网格分辨率的结论

论文比较了 `100 nm`、`200 nm`、`500 nm` 三种中间图像步长。

结论是：

- 靠近表面、Sentaurus 网格很细时，小步长更准确
- 深层区域如果中间图像步长太小，而 Sentaurus 网格太粗，反而会出现部分数据没有被读取的问题
- 对他们的 2D 电子探测器问题，`200 nm` 是较好的折中

因此项目实现时应注意：

- 不是步长越小越好
- 中间图像分辨率要和 Sentaurus 网格尺度匹配

## 5. 3D 扩展流程

论文还给出了 3D 扩展版本，主流程不变，只是维度从 2D 扩成 3D。

3D 时的变化有：

- 中间图像由 2D 规则网格变成 3D 规则网格
- 需要指定三个方向的尺寸与步长
- 保存时可先 reshape 成 2D 文本格式
- 读回后再恢复为 3D 数组
- `interpn` 继续用于三维插值

论文也指出 3D 情况下更容易出现深层区域漏读，因为 substrate 的 Sentaurus 网格通常比 pixel 附近粗得多。

## 6. 论文验证流程

论文不是只做了转换，还做了两层验证。

### 验证 1：Monte Carlo 分布 vs Sentaurus Optical Generation

- 比较 Casino 输出的电子分布
- 与插值后写入 Sentaurus 的 `OpticalGeneration` 分布对比
- 检查不同深度区域中生成电子数是否一致

### 验证 2：TCAD 模拟结果 vs 实验结果

- 利用 9 像素结构扫描电子束位置
- 得到 `PSF`
- 对 `PSF` 做傅里叶变换得到 `MTF`
- 再与 TEM 实验测得的 `MTF` 比较

论文结论是该方法能够较好复现实验趋势。

## 7. 这篇文献对应到本项目时，Python 需要承担的任务

如果映射到当前项目，Python 侧至少要完成以下功能：

1. 读取 Monte Carlo 输出文件
2. 构建规则均匀网格的 intermediate picture
3. 完成电子数或能量沉积到生成率的换算
4. 读取 Sentaurus 网格坐标
5. 用 `scipy.interpolate.interpn` 做 2D 或 3D 插值
6. 生成可回写到 Sentaurus 的数据文件

## 8. 对当前项目最重要的实现点

对当前代码最关键的不是完整复刻 Sentaurus 全流程，而是先把论文中的 Python 核心做对：

1. 规则网格的构建方式
2. 中间图像数值累积方式
3. 单位换算
4. `interpn` 插值
5. 输出格式与 Sentaurus 网格点的一一对应

## 9. 论文方法的局限

论文也明确给出了几个局限：

- 中间图像只用单一固定步长时，深层区域可能漏掉部分粒子
- 3D 情况下这个问题更明显
- 该方法导入的是静态分布，不是时间传播轨迹
- 因此不适合做粒子在器件中的时序传播模拟

## 10. 一句话总结

这篇论文的方法链条可以概括为：

`Casino/Geant4 粒子输运`  
`-> Python 构建均匀 intermediate picture`  
`-> Python + scipy.interpn 插值到 Sentaurus 网格`  
`-> Python 重写 OpticalGeneration 数据`  
`-> TDX 重新封装为 tdr`  
`-> Sdevice 做器件响应仿真`
